[教學] 用 NodeJS 編譯 Blog 到 Github (一)

2013/10/2
NodeJSJavascript

因有許多朋友有興趣並詢問如何在 GitHub 搭建自己的部落格, 這邊我會把自己建站的過程與資訊一併整理出來

為什麼要把部落格建立在 Github 上

現今網路上已有許多的免費部落格： PIXNET, Xuite, Blogger, WordPress... 等
很多人也許不明白，這樣做的好處是什麼，以下列舉幾項我認為特別突出的優點：

渾然天成的 Git 版本控制

有在寫 code 的人一定都明白版本控制的重要利與便利性
光是這點，我看就可打趴所有的主流部落格營運商了:)

易於備份的文章

看著最近 "Google Reader" 與 "無名" 吹著熄燈的號角，就不禁感嘆天下無不散的宴席
我們果然不能太依賴免費服務阿
所以這樣建站的好處是所有的文章與原始碼全都存放在自己 local 端上
未來如果營運商宣布不經營的話，隨便找台主機再 update 上就又復活囉

可在自己最愛的 Edtor 上編寫

一般線上的部落格，發表文章都是用他們自己的後台系統
這樣造成的不便是可想而知的，每次發表與更新都要進到繁鎖的管理系統
實在是非常沒效率阿 :(
而這樣的反差在自己編譯部落格後儼然成為一種強烈的對比
編輯器打開後隨寫隨存 :)
若再搭配 Dropbox 類的同步軟體，就可以在任何地方同步

完全客製化

正所謂「我的部落格我作主」
一般線上部落格雖然有提供幾套 themes 可選，但很難找到自己喜歡的
或是很容易跟別人的主題一模一樣變得沒特色
雖然很多後台都有提供 CSS 修改，但還是一樣要繁鎖的進入管理介面去做修改
再說我相信一個有追求與潔癖的工程師都會很難忍受，頁面上有任何不屬於自已的 code

超酷的 domain

現今 GitHub 已經成為工程師的身份証 ID
我個人也非常喜歡 your_id.github.io 的形式 :)
又可免費搭上愈來愈夯的 .io 域名列車
但如果真的不喜歡 Github 的 domain 要換上自己的也是 ok 的拉

優點就大概介紹到這邊，下一章節會正式進入實作
