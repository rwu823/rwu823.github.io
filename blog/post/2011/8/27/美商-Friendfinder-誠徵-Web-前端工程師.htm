指尖上的前端

美商 Friendfinder 誠徵 Web 前端工程師

2011/8/27
Life

敝公司要強力擴大團隊囉，Web Front-End Engineer(F2E 前端工程師) 我認為這是一個最難找到"合適"人選之一的位置， 基本上 F2E 是因為 Internet 的興起與快速發展所誕生出來的，傳統製做網站的過程中，功能與需求並沒有太複雜， 所以往往都是設計 Layout 完頂多再切個版就到後端套程式了，但如今網際網路的應用與發展已經不可同日而語了， 需求的是更迅速地與使用者互動應對與更具親和更有趣的用戶體驗。

有興趣的人，更可以看看 Google 製作的 Internet 與 Browsers 發展史， 如果看完後你覺得熱血沸騰，那恭喜你可以繼續往下跳入火坑了 :)

好拉進入重點，個人認為前端工程師最重要的二個特質：

紮實的觀念

知道何謂前端工程？前端工程師該做什麼？知道自己要什麼？技術的廣度與深度，你會的是什麼有多深。不要認為前端工程師只要懂"前端"就足夠了。

不死的熱情

坦白說，我覺得前端工程師還蠻累的，要有耐心的處理網頁上px間的差距，還要細心的觀察細微的事物，需要關注的技術很多又很廣。所以一定要熱愛前端、有呈現完美瀏覽體驗給用戶跟確保代碼建壯發展交付到後端的使命，這樣才能從這份工作上面找到成就感。

我們的前端工程師有分為二個方向，桌面跟移動網站(Mobile)開發，二個方向所要求的技術與專精也不太相同，傳統的桌面網站開發比較注重的是跨瀏覽器的兼容實現與團隊的溝通協作等。而 Mobile 網站的開發，更注重的是新技術的關注與實踐。二邊都非常刺激有趣:b
